CDL Persistent Ids Must…
•Identify, whether or not the object is at hand
–It may not be convenient, helpful, or permitted for you to inspect the object itself -- metadata needed
•Convey different flavors of permanence
•Lead to access (if authorized)
–Not strictly an “identification” problem, but it is the “404 not found” that we need to fix
•Be valid for some longish period
•Be carried on, in, or with the object
All of these are service issues, but the first three can be shored up a little by the identifier scheme.
